W = Mundo
I = Interfaz
M = Máquina

Sucesos relevantes:

W:
-Usuario va a buscar una bicicleta a una estación de la periferia.
-Usuario va a buscar una bicicleta a una estación del centro.
-Usuario se identifica en una estación.
-Empleado rechaza solicitud de retiro de bicicleta debido a penalización.
-El usuario retira una bicicleta de alguna estación
-El usuario devuelve una bicicleta en alguna estación
-Una estación no dispone de bicicletas para retirar
-El usuario nunca devuelve una bicicleta
-Usuario (que ya dispone de una) solicita otra bicicleta.
-Usuario se desplaza entre estación y estación.
-Alguien se hace pasar por un usuario para retirar/devolver una bicicleta
-Los camiones de la empresa de transporte proveen una estación con más bicicletas.
-Los camiones de la empresa de transporte retiran bicicletas de una estación.
-Se añade una nueva estación a la red.
-La gente se traslada ida y vuelta a su trabajo/casa en una franja horaria reducida, llamada "hora pico".
-La estación no abre.
-El usuario sufre un accidente.
-La bicicleta sufre un desperfecto.
-El usuario rompe una bicicleta
-Usuario olvida contraseña
-Usuario espera por bicicleta
-El usuario devuelve una bicicleta que no está en condiciones.
-El usuario devuelve una bicicleta fuera de horario.
-El usuario devuelve una bicicleta en una estación cerrada
-Ninguna estación dispone de bicicletas
-Un nuevo usuario no provee la información necesaria para inscribirse.
-Empleado le entrega una bicicleta a un usuario que no está registrado.
-Empleado le entrega una bicicleta a un usuario que está penalizado.
-Gobierno entrega más bicicletas a la empresa de transportes.
-Sistema pierde la conexión.
-La empresa de transporte no cumple con el tiempo estimado en trasladar las bicicletas.
-Una estación queda aislada
-La empresa de transporte no se puede movilizar.
-Muchos usuarios se registran a la vez.


I:
-El sistema indica a la compañia de transporte que una estación requiere/tiene exceso de bicicletas y le indica desde dónde y hacia dónde desplazarlas
-Administrador realiza consulta de estadísticas relevantes al sistema (Falta en el diagrama de contexto)
-La app envía consulta al sistema
-La app envía datos de registración de usuario al sistema.
-El sistema le responde consulta a la app.
-Empleado solicita reposición o notifica exceso de bicicletas en una determinada estación
-Empleado envía información de autenticación de un usuario para ser validada por el sistema.
-Sistema devuelve al empleado el resultado de autenticación válida o inválida
-Empleado ingresa modificación de stock de bicicletas para una determinada estación.


M:
-El sistema calcula el mejor conjunto de estaciones que disponen de bicicletas para otra estación.
-El sistema actualiza el estado de un usuario.
-El sistema detecta una discrepancia entre la solicitud de un usuario y el estado del usuario en el sistema.
-El sistema registra estadísticas relevantes
-El sistema actualiza el estado de una estación.
-El sistema recalcula distribución de bicicletas cuando se informa la caída de conexión con una estación.
-El sistema recalcula distribución de bicicletas cuando se agrega una estación a la red.
-El sistema recalcula distribución de bicicletas cuando se elimina una estación de la red.
